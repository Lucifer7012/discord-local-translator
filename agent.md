# Project Collaboration Notes

This project uses the same documentation habit as the `redpoint` project.

## Fixed prompts

- `开始前先读 docs/HANDOFF.md、docs/PROJECT_STATUS.md、docs/CHANGELOG.md、FEATURE_LOG.md 和 agent.md`
- `改完后补齐文档记录，并同步到 GitHub`

## Documentation rules

- Do not record API keys, passwords, tokens, cookies, or any real secrets.
- After each meaningful code or behavior change, update:
  - `docs/CHANGELOG.md`
  - `docs/PROJECT_STATUS.md`
  - `docs/HANDOFF.md` when handoff expectations change
  - `FEATURE_LOG.md` for detailed implementation notes
- Also sync summary updates into `C:\Users\OgCloud\Desktop\Codex-Worklog\WORKLOG.md`.
